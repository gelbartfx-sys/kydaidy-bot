## Ponytail: режим ленивого сеньора (анти-переусложнение)

**Лестница решений** (остановись на первой ступени, которая держит):
1. Это вообще надо строить? (YAGNI)
2. Это уже есть в кодовой базе?
3. Это делает стандартная библиотека?
4. Это покрывает нативная фича платформы?
5. Это решает уже установленная зависимость?
6. Можно ли в одну строку?
7. Только теперь: пиши минимум кода, который работает.

**Багфиксы:** отчёт называет симптом. Найди всех вызывающих функцию, которую трогаешь, и почини общую функцию один раз — один guard там меньше, чем по одному на каждый вызов. Лечи причину, не симптом.

**Правила:**
- Никаких незапрошенных абстракций, зависимостей, boilerplate
- Удаление важнее добавления; скучное важнее умного; минимум файлов
- Побеждает кратчайший рабочий diff — но только после понимания задачи
- Оспаривай сложные запросы; при равном размере корректность к краевым случаям важнее «умного»
- Помечай намеренные упрощения комментарием `ponytail:` — что упростил и куда смотреть, если усложнять

**Не режется никогда:**
- Полное понимание задачи до кода
- Валидация ввода на границах доверия
- Обработка ошибок, предотвращающая потерю данных
- Безопасность, доступность
- Всё, что явно запрошено
- Нетривиальная логика требует ОДНОЙ запускаемой проверки (assert/маленький тест)
