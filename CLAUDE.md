# CLAUDE.md — память проекта kydaidy-bot

> Telegram-бот воронки «kydaidy / куда иди?»: тест архетипов Тени → бесплатная
> AI-встреча с «Алёной» → продажи (Клуб «Манифест», воркбук «Манифест 7», 1:1).
> Python + aiogram 3 + aiohttp, деплой на Render (Frankfurt), БД — Cloudflare D1
> с фолбэком на локальный SQLite. Репозиторий **публичный** — ключи только в env.

Язык проекта и коммитов — **русский**. Тон бренда — «анти-лайфкоучинг»: лещ
честности + механизм, без давления, дефицита, эзотерики, «5 шагов» и обещаний
результата. Держать этот тон во всех текстах (Алёна, дожим, куратор, growth).

## Запуск и деплой

- Локально: `python bot.py` (нужен `.env` из `.env.example`, минимум `TG_BOT_TOKEN`, `TG_ADMIN_ID`).
- Render: `render.yaml` (web-сервис, `python bot.py`, healthcheck `/health`, autoDeploy). Секреты (`sync: false`) вводятся в Render UI.
- `.github/workflows/keep-alive.yml` пингует `/health` каждые 14 мин (free-tier Render засыпает через 15).
- Тесты (чистая логика, без сети/БД): `python3 test_lead_policy.py`, `python3 test_brain.py`.

## Порядок роутеров в bot.py (критично)

Текст-фильтры перехватывают сообщения раньше catch-all fallback:
`curator_router` (режим правки Алёны) → `alena_router` (активная встреча) →
`growth_router` (только callback) → основной `router` (handlers) + `guide_router`.
При добавлении роутера с текст-фильтром — учитывай этот порядок.

## APScheduler-джобы (в bot.py)

`run_nurture_tick`, `run_followup_tick`, `run_stale_session_tick` (мягкий оффер
на затихшей встрече), `run_credit_check`, `run_growth_tick`, `push_daily_batch` +
`publish_tick` (куратор). Периоды — во флагах config.

## Внешние сервисы

- **Telegram** (aiogram 3), webhook `webhook_path` (`/tg-webhook`).
- **Gemini** (`GEMINI_KEY`) — тест Тени, тексты Алёны/куратора/growth, распознавание видео-кружков от клиента.
- **HeyGen** (`heygen_api_key`) — TTS-голос Алёны (бесплатно) + живые видео-кружки (тратят кредиты).
- **Anthropic** (`ANTHROPIC_API_KEY`, SDK читает сам) — мозг v2 (Opus 4.8 диагноз, Sonnet ответ).
- **Tribute** — платежи/подписки (webhook). **Tally** — захват из квиза (webhook).
- **Cloudflare D1** — БД (через Pages-прокси `D1_PROXY_URL` или CF REST API).
- **Calendly** — запись на 1:1 (`calendly_1on1_url`).

## Инварианты / правила

- Всё, что зовёт внешний API (голос, кружки, мозг, кредиты), должно быть **крэш-сейф**: сбой → фолбэк, воронка не падает.
- Ключи **никогда** не хардкодятся (репо публичный) — только env.
- Бот не пишет юзерам сам без явного одобрения (growth = режим ревью через Кая).
- Webhooks fail-closed: без настроенного секрета подпись отклоняется (`webhook_dev_allow_unsigned` только для локалки).
- Экономика кружков: бюджет HeyGen-кредитов = % от цены Клуба по треку лида (`lead_policy`).

## Последняя проделанная работа (свежие коммиты)

- **Мурадян-арсенал в диагноз**: 7 приёмов живой сессии (защита без давления, вторичная выгода, слово↔тело, канал тела, претензия=чувство, пауза-присутствие, незавершённое предложение), бренд-фильтр применён.
- **Аудит A1+A2 закрыт**: идемпотентность оплаты (дедуп `payment_id` через `bot_meta`, метка после гранта); атомарное открытие встречи (`INSERT WHERE NOT EXISTS`) + закрытие всех активных; `bot_meta` добавлен в SQLite-схему (паритет с D1).
- **Голос железно-2**: потолок 500 токенов на голосовой ход, TTS-гейт 900→1600 (длинная реплика уходит голосом, не текстом), телеметрия `voice_fallback_text`.
- **Стыки по-человечески**: текст-дублёр после голосового ротируется; инструкция только в первых 2 ходах; голосовое после кружка состыковано (не здоровается заново).
- **Волна 3**: кнопки первого шага (барьер чистого листа), голосовое приветствие с именем, чекпойнт пути на 5-м ходу.
- **Мандаты Кая**: оффер = именной КРУЖОК (тизер голосом → рендер твина → кружок+кнопка, страховка текстом); кружки ОТ клиента распознаются (Gemini video); отработка возражений после оффера (до 3); голос = канал по умолчанию.
- **Волна 1**: голос Алёны в диалоге и оффере (HeyGen TTS), события воронки `funnel_events` + `/sources`, дожим 3 касания с мягким дедлайном.

Детальный контекст решений — в `docs/hermes/` (ai-coach-architecture, credit-alerts-SPEC) и `docs/` позиционирования/тона (voice-and-turns-sharp), на которые ссылаются докстринги (сами docs могут жить вне этого репо).
