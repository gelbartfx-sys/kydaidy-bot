## 2. БЛОК 1 — Peak-баг догона
**Scope:** на пике боли (`is_peak`) порог тишины перед stale-нуджем РАСТЁТ, не падает.
**Якоря:** `config.py:172-175` (`stale_nudge_peak_minutes`), `alena_chat.py:1906-1926` (`run_stale_session_tick`).
**Баг:** единственная проверка `idle` привязана к `if not is_peak`. Пиковая встреча не перепроверяется по времени → нудж летит на 7-й минуте (`peak_min=min(7,20)=7`).
**Фикс:**
- `config.py:175`: `stale_nudge_peak_minutes` = **35** (нормальный 20 + запас 15). Комментарий переписать: пик = ЗАМЕДЛЯЕТ решение.
- `alena_chat.py:1917-1926`: заменить асимметричный `if not is_peak:` на симметрию для обеих веток:
  `threshold = peak_minutes if is_peak else normal_minutes; if idle is None or idle < threshold: continue`
**Safety:** не трогать `reengage_minutes`/`stale_nudge_minutes`; `dead_session_tick` (30 мин после `nudged_at`) не зависит от минуты нуджа — перепроверить глазами.
**Приёмка:** пиковая встреча idle 8-34 мин → нудж НЕ уходит; idle≥35 → уходит как раньше; обычная встреча без изменений.
**Rollback:** вернуть `=7` + асимметричную проверку. Без миграций.

Вызов `refresh_purchase_stage(tg_id)`: (1) `alena_chat.py:1268` после `save_client_model` (каждый ход); (2) `handlers.py:174` после `subscribe_confirmed`; (3) `handlers.py:718` после `buy_click`. Событийно, без cron.

**6.3 Обёртка гейта (fail-open):**
```
async def stage_allows_offer(tg_id) -> bool:
```
